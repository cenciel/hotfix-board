# SOOP 핫픽스 대시보드

Jira 릴리즈 데이터 기반으로 SOOP Android 앱의 핫픽스 배포 패턴을 분석하고, 이번 달 핫픽스 발생 가능성을 예측하는 내부 대시보드입니다.

🔗 **[hotfix-board.streamlit.app](https://hotfix-board.streamlit.app)**

---

## 주요 기능

- **월별 릴리즈 현황** — 핫픽스 / 정식 릴리즈 누적 막대 및 선형 차트
- **연도별 월평균 핫픽스** — 연도 단위 트렌드 비교
- **이번달 핫픽스 전망** — 동월 과거 이력 + 최근 3개월 추세 가중 계산
- **릴리즈 로그** — 전체 배포 이력 테이블 (Jira 링크 포함)
- **데이터 로드 버튼** — 누르는 시점에 Jira에서 최신 데이터 병합

---

## 파일 구조

```
hotfix-board/
├── app.py              # Streamlit 대시보드 + Jira 연동 로직
├── config.toml         # Jira 접속 설정 (토큰 제외)
├── requirements.txt
├── .env                # API 토큰 (로컬 전용, Git 제외)
├── .gitignore
└── data/
    └── releases.json   # 누적 릴리즈 데이터
```

## 데이터 업데이트

대시보드 사이드바의 **📥 데이터 로드** 버튼을 클릭하면 Jira에서 Android 릴리즈 목록을 가져와 `releases.json`에 병합합니다.

- 기존 데이터는 삭제되지 않고 신규 버전만 추가
- 로드 결과는 사이드바에 즉시 표시

---

## 핫픽스 판별 기준

버전 마지막 세그먼트가 `0`이면 정식 릴리즈, `1` 이상이면 핫픽스로 분류합니다.

| 버전 | 분류 |
|------|------|
| `8.24.0` | 🟢 정식 릴리즈 |
| `8.24.1` | 🔴 핫픽스 |
| `8.24.2` | 🔴 핫픽스 |

Jira `description` 필드에 `hotfix` 텍스트가 포함된 경우에도 핫픽스로 분류됩니다.

## 이번달 예측 계산식

```
핫픽스 발생 확률 = (동월 과거 이력 비율 × 0.6) + (최근 3개월 비율 × 0.4)
```

- **동월 과거 이력** — 과거 같은 달에 배포된 릴리즈 중 핫픽스 비율
- **최근 3개월** — 직전 3개월간 전체 릴리즈 중 핫픽스 비율
